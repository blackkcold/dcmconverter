# ADR 0002: Use SDD And TDD / 使用 SDD 和 TDD

## Status / 状态

Accepted / 已接受

## Decision / 决策

Feature 级别先写规格，模块级别用测试驱动实现，UI 用验收测试补充。<br>
Write specs at the feature level, implement modules with test-driven development, and supplement UI coverage with acceptance tests.

## Consequences / 影响

需求可追踪，回归风险更低；初始开发成本更高。<br>
Requirements stay traceable and regression risk is lower; the initial development cost is higher.
