# Security And Privacy / 安全与隐私

## 本地处理 / Local Processing

项目不设计上传接口。浏览器只能读取用户通过文件选择器或目录选择器授权的文件。<br>
The project does not expose an upload path. The browser can only read files explicitly authorized through the file or directory picker.

## 默认匿名 / Default Anonymization

默认导出选项开启 `anonymizeOverlay`。PatientName 显示 `Anonymous`，PatientID 显示 `Hidden`。<br>
The default export option enables `anonymizeOverlay`. PatientName is shown as `Anonymous`, and PatientID is shown as `Hidden`.

只有用户显式开启个人信息模式时，导出的 overlay 才会显示原始患者信息。<br>
Original patient information is shown in the exported overlay only when the user explicitly enables personal-info export mode.

## 命名保护 / Naming Protection

导出包名称会清洗非法字符；JPEG 文件名模板只开放主字段白名单，不包含 PatientName / PatientID / PatientSex / PatientAge / BirthDate / AccessionNumber / InstitutionName 等敏感字段。序号前缀和重名兜底后缀由系统生成。<br>
The export package name is sanitized for invalid characters; JPEG filename templates only expose the main-field whitelist and do not include sensitive fields such as PatientName / PatientID / PatientSex / PatientAge / BirthDate / AccessionNumber / InstitutionName. Sequence prefixes and duplicate-name fallback suffixes are generated by the system.

导出报告和断点续传 manifest 不保存原始源目录结构；这可以避免泄露用户按患者姓名、机构或检查组织的本地路径。<br>
Export reports and resume manifests do not store the original source directory structure, reducing leakage of local paths organized by patient name, institution, or exam.

## 风险提示 / Risk Note

V1 不自动识别或清除像素内烧录的患者信息。用户导出前需要确认图像内容。导出面板底部常驻显示此风险提示。<br>
V1 does not automatically detect or remove burned-in patient information. Users must confirm the image content before exporting. This risk note is permanently displayed at the bottom of the export panel.

## PWA 缓存边界 / PWA Cache Boundary

PWA Service Worker 只预缓存构建期应用壳资源，包括 HTML、JS、CSS、图标、manifest、Cornerstone worker 和 WASM 解码器产物。用户通过文件或目录选择器授权的 DICOM 文件、File 对象、blob URL、file: URL、导出 JPEG/ZIP 和 File System Access handle 不会写入 Cache Storage、IndexedDB、localStorage 或 sessionStorage。<br>
The PWA service worker only precaches build-time app-shell assets, including HTML, JS, CSS, icons, the manifest, Cornerstone workers, and WASM decoder build outputs. User-authorized DICOM files, File objects, blob URLs, file: URLs, exported JPEG/ZIP data, and File System Access handles are not written to Cache Storage, IndexedDB, localStorage, or sessionStorage.

Service Worker 更新采用提示模式。新版本可用时必须由用户确认后才激活；应用不会因后台更新自动刷新并丢失当前内存态。<br>
Service worker updates use prompt mode. A new version is activated only after user confirmation; the app does not automatically reload on background updates and lose current in-memory state.
