# F4 功能规格：JPEG 导出 / F4 Functional Spec: JPEG Export

## 目标 / Goal

把当前 viewer canvas 导出为 JPEG；批量结果可打包 ZIP，并支持共享导出包名称与受控文件名模板。<br>
Export the current viewer canvas as JPEG, package batch results into ZIP when needed, and support a shared export package name plus controlled filename templates.

## 输入 / Input

`HTMLCanvasElement`、`DicomMetadata`、`ExportOptions`。<br>
`HTMLCanvasElement`, `DicomMetadata`, and `ExportOptions`.

`ExportOptions` 包含导出包名称、文件名模板模式、预设模板和受控字段组合。<br>
`ExportOptions` includes the export package name, filename template mode, preset templates, and controlled field combinations.

## 输出 / Output

`JpegExportResult` 或 ZIP `Blob`。<br>
`JpegExportResult` or a ZIP `Blob`.

folder export 的根目录、ZIP 内路径前缀和 ZIP 下载名都使用同一个导出包名称。<br>
The folder-export root directory, ZIP internal path prefix, and ZIP download name all use the same export package name.

## 验收标准 / Acceptance Criteria

- JPEG 使用 `canvas.toBlob('image/jpeg')`。<br>JPEG uses `canvas.toBlob('image/jpeg')`.
- 导出包名称会同时驱动 folder export 根目录、ZIP 内路径前缀和 ZIP 下载名。<br>The export package name drives the folder-export root directory, ZIP internal path prefix, and ZIP download name.
- 文件名模板只暴露主字段白名单，不支持自由文本拼接。<br>The filename template only exposes the main-field whitelist and does not allow free-text composition.
- 序号前缀和重名兜底后缀仍由系统自动补齐。<br>The sequence prefix and duplicate-name fallback suffix are still generated by the system.
- ZIP 打包失败时返回 `ZIP_EXPORT_FAILED`。<br>When ZIP packaging fails, `ZIP_EXPORT_FAILED` is returned.
- 单个导出失败可被任务状态记录。<br>Individual export failures can be recorded in task state.
